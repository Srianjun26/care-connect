[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through our workflow architecture. (pause)
First, the patient query starts at the START node and flows into the Security Checkpoint. (pause) The Security Checkpoint scrubs PII and checks for prompt injections.
If the check passes, the flow moves to the Orchestration Node, which invokes the Care Orchestrator agent.
The Care Orchestrator agent delegates symptoms to the Symptom Triage Agent to assign a severity, and then delegates to the Doctor Comm Agent to compile a clinical note draft.
Next, the workflow enters the Human Approval Node, which uses the ADK Request Input tool to pause the graph and ask the reviewing physician for approval.
Finally, when approved, the flow goes to the Final Output Node, which calls our local MCP Server's save_clinical_note tool to commit the note to the patient's record.
